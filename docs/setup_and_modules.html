Dev Environment

Install the IDE

Our preferred IDE is STM32Cube. This is because one can easily configure output and input pins of the NUCLEO board. Click here to install.

Create a workspace directory somewhere on your computer. This is where the IDE will search for your projects.

Inside that directory, create a folder for the Connect 4 project.

GitHub Repository

The link to the GitHub Repository is here. Initialize a Git repository in your Connect 4 project folder using an app like Git Bash or GitHub Desktop, add the GitHub Repository as a remote origin, and pull the contents of the GitHub repository into that project folder.

Modules

Most of the code we are going to be editing is in the "{Project Directory}/Core/Src" folder.

main.cpp

This module will serve as the entry point into our application. The main bulk of our code will be in the "while(1)" loop.

kernel.h

This module checks for input signals on the NUCLEO board.

run_kernel()

Takes no arguments and returns an integer.

Currently loops through the states list and returns the x coordinate of the "target" that was hit. If there is no target hit, it returns -1.

y_getter.h

get_y(int x)

Takes the x coordinate (as an integer) of the target hit. Returns an integer y based on the number of pieces in column x.

During this function, the list that tracks the height of each column will have the x column be incremented by 1.

win_checker.h

This module tracks the players turn and the location of their pieces. This information is used to check if a player has won.

change_turn()

Takes no arguments and returns void.

Changes the turn_number from 1 to 2 or vice versa. Changes the turn_player_locations variable to hold a reference to the list that holds the locations of the current player's pieces.

check_win(list[int] location)

Takes a list (of integers) of the position of the piece being "inserted" into the board. Returns a Boolean if the current player won.

Inserts the location parameter into the turn_player_locations list and checks in all directions relative to the new pieces to see if four pieces are connected (the player won).
